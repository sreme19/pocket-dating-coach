# Profile Page — Design Brief
**For:** Claude Design / Redesign Review  
**Route:** `/verified-vibe/profile`  
**Purpose:** This file documents every section on the Profile page as it stands today — layout, content, data source, empty states, interactions, and known design problems. Use this as the source of truth for a redesign.

---

## Page Structure

The page has two tabs at the top:

```
┌────────────────────────────────────────────────┐
│  ← My Profile                          [share] │
│                                                │
│  [ ❤ The Public Read ]  [ ⚡ Trust & Boost +21] │
└────────────────────────────────────────────────┘
```

- **The Public Read** — what potential matches see
- **Trust & Boost** — owner-only view; proof uploads and trust score

Both tabs are scrollable, mobile-first, dark-themed (background `#0B1120`), max-width ~430px.

## Section 1 — Hero

**Purpose:** First impression. Full-bleed photo with identity overlay.

**Current layout:**
- Full-width image at `aspect-ratio: 4/5` (portrait crop)
- Dark gradient overlay fades from transparent (top) to near-black (bottom)
- Identity block sits at bottom-left over the gradient

**Content displayed:**
- Lead photo (from Supabase `avatar_url` or first localStorage photo)
- Name (italic serif, 26px, white) — editable via ✏️ pencil icon
- Age (same row as name)
- City (12px, 70% white opacity, with MapPin icon) — editable
- Archetype badge (small pill, green border, uppercase, e.g. "CASUAL GENEROUS")
- Trust score pill (green tinted, e.g. "+21")
- Pencil edit button (top-right of identity block) — opens inline name/age/city form

**Data source:** `verified_vibe_users` table → user store (`$user.firstName`, `$user.age`, `$user.city`, `$user.avatar`)

**Empty state:** Grey placeholder with 📸 camera emoji + "Add photos to boost your profile"

**Interactions:**
- Tap pencil → inline form with 3 inputs (name, age, city), save button
- Share button (top-right of page header) → share profile URL

**Mobile:** Full width, fixed aspect ratio. Identity text scales slightly at 768px+.

**Design issues:**
- Name/age/city edit is a small floating form; no visual affordance that it's editable
- Archetype badge and trust pill compete for space in the name row
- No way to change the lead photo from this page (must go to verification)

## Section 4 — Archetype Q&A (Chemistry / Dating Style)

**Purpose:** Shows the user's archetype-specific answers — vibe, chemistry, lifestyle preferences, values.

**Current layout:**
- Section label = archetype name (e.g. "Casual Generous Man") with info icon
- Vertical list of Q&A items, each with: icon | label | value (chip row or plain text)
- Each item has a thin top border separator

**Content displayed:**
- 4–7 rows depending on archetype, e.g.:
  - 🔥 Chemistry — "Slow Burn"
  - 💫 Relationship energy — "Light, consistent"
  - 🏡 Lifestyle — "Active social calendar"
  - ❤️ Values — "Loyalty, adventure"
  - "Self declared" badge on self-reported items

**Data source:** `archetypeSections` array built from `vv_casual_generous_profile` + `vv_qa_responses` localStorage keys, and Supabase `master_profile`

**Empty state:** Section hidden entirely if no archetype QA data

**Interactions:**
- Pencil icon → navigates to `/verified-vibe/verification?step=archetype_qa&returnTo=/verified-vibe/profile`

**Mobile:** List items stack cleanly.

**Design issues:**
- Navigating away to edit disrupts the profile editing flow
- "Self declared" label is small and easy to miss
- Section label using archetype name is confusing — it looks like a heading but isn't descriptive to the viewer

## Section 5 — Money Matters

**Purpose:** Financial credibility signal — income, net worth, spending patterns.

**Current layout:**
- Section label with 💰 emoji + "Money Matters" + pencil edit for chips
- Gold-tinted card (`border: 1px solid rgba(212,175,55,0.3)`)
- Inside the card, up to 3 sub-components:

  **A. Income/net worth stats row** (if self-declared):
  - Side-by-side stat blocks: icon + label ("Annual Income") + value ("1Cr+") + "SELF DECLARED"

  **B. Wealth insight signal-grid** (if bank/ITR uploaded):
  - 3-column grid of emoji tiles (e.g. ✈️ Frequent Traveler, 📈 Active Investor)
  - Aggregated sentence below chips in italic
  - "✓ AI verified via bank statement / financial document" footer
  - In edit mode: × remove button per tile + "✨ Suggest 3 more"

  **C. Spending breakdown chart** (if receipts uploaded):
  - Category rows: emoji + category name + amount label + horizontal fill bar
  - "✅ Receipt verified" or "🏦 Bank statement" badge

- Nudge CTA if wealth proof exists but no receipts: "🧾 Upload receipts to show spending patterns →"

**Data source:** `vv_money_matters` (self-declared) + `vv_proof_insights[category='wealth']` + `vv_proof_insights[category='spending']`

**Empty state:** "💳 Set this up in Trust & Boost →" button (navigates to boost tab)

**Interactions:**
- Pencil icon on section label → edit chip mode (remove chips, suggest more via AI)
- Nudge CTA → `/verified-vibe/proof-upload?category=spending`

**Mobile:** Spending bar chart scrolls within card if many categories.

**Design issues:**
- Card has a lot of density when all three sub-components are visible
- Spending bar chart feels grafted on — different visual language from the chip grid above it
- "SELF DECLARED" label is styled dimly; easy to mistake declared income for verified

# Cross-Cutting Design Problems

These issues span both tabs and should be addressed holistically:

1. **Section sameness** — Lifestyle Signals, Health & Fitness, Social Life, Career Highlights, and Money Matters all use similar chip/tile treatments. By section 9, the visual rhythm is monotonous.

2. **Edit mode inconsistency** — Some sections use a pencil toggle (inline edit), some navigate to a new page, some open modals. No consistent editing pattern.

3. **Empty section invisibility** — Sections with no data simply vanish. New users see a nearly empty profile with no explanation of what could be there or how to add it.

4. **Trust & Boost ↔ Public Read disconnect** — The connection between uploading proof and seeing it on the profile is not visually explained. Users must discover it.

5. **No progress indicator** — There is no overall "profile completeness" percentage or checklist showing what the user has and hasn't filled in.

6. **AI actions are scattered** — AI-generate buttons exist on 8+ different sections but there is no unified "✨ Enhance my whole profile" entry point.

7. **Mobile typography** — Section labels, chip labels, and bar chart text can all be hard to read on OLED dark backgrounds at small sizes.

8. **Portrait placement** — The two AI portrait sections break the natural content flow (personality → career → lifestyle → garage). They would feel more intentional if grouped or placed at a single "gallery" position.
